## Design Context

### Users
Solo admin (the developer themselves). This is a personal internal tool — no onboarding needed, no need to explain affordances. Efficiency and density trump discoverability. The admin opens this to fix scores and set up bracket matches, then closes it. Speed of operation matters more than anything decorative.

### Brand Personality
Precise, athletic, operational. Think a match official's digital clipboard or a live-broadcast statistics terminal. Three words: **disciplined, immediate, authoritative**.

### Aesthetic Direction
Dark mode — navy-black backgrounds, cyan accent (#00d4ff), Barlow Condensed for display/labels, Barlow for body. Sports-data aesthetic: dense rows, clear hierarchy, status expressed through color. The mission-control feel is intentional — keep it, refine it. Not a consumer app; not a showcase. A sharp tool.

Anti-references: generic SaaS dashboards with white backgrounds and rounded corners everywhere; colorful marketing pages; anything that feels like it was made to impress non-users.

### Design Principles
1. **Density over whitespace** — this is a data tool; information should be legible at a glance, not padded for aesthetics
2. **Color as signal** — the stage colors and status colors carry meaning; never decorate with color for its own sake
3. **Every state is handled** — hover, focus, loading, empty, error — all states are explicit, never invisible
4. **Operational trust** — the UI should feel like it does what you tell it, instantly and without drama
5. **Typography does the hierarchy** — Barlow Condensed uppercase for labels and IDs; Barlow regular for values; weight/size/color do the rest
